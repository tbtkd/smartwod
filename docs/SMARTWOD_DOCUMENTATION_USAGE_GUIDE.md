
# DOCUMENTATION_USAGE_GUIDE.md
Guía para Usar y Mantener la Documentación de SMARTWOD

Este documento explica cómo utilizar y mantener actualizada
la documentación del proyecto.

---

# 1. Objetivo

La documentación permite:

- entender rápidamente el proyecto
- facilitar mantenimiento
- permitir incorporación de nuevos desarrolladores
- preservar decisiones técnicas

---

# 2. Archivos de Documentación

El proyecto utiliza los siguientes archivos principales:

README.md

Descripción general del proyecto.

SMARTWOD_PROJECT_INDEX.md

Índice maestro que conecta toda la documentación.

PROJECT_STRUCTURE.md

Explica la estructura del código.

AI_TECHNICAL_CONTEXT.md

Describe el contexto técnico del sistema.

ARCHITECTURE_DECISIONS.md

Explica decisiones de arquitectura.

WORKOUT_ENGINE.md

Documenta el motor del timer.

ROADMAP_PRODUCT.md

Describe evolución del producto.

MONETIZATION_STRATEGY.md

Describe estrategia de monetización.

CONTRIBUTING.md

Guía para colaboradores.

---

# 3. Orden de Lectura Recomendado

Para comprender el proyecto:

1. README.md
2. PROJECT_STRUCTURE.md
3. AI_TECHNICAL_CONTEXT.md
4. WORKOUT_ENGINE.md
5. ARCHITECTURE_DECISIONS.md

Después de estos documentos se entiende completamente el sistema.

---

# 4. Cuando Crear Nueva Documentación

Crear un nuevo archivo cuando:

- se agregue una arquitectura nueva
- se introduzca un sistema complejo
- se cambie el motor del entrenamiento
- se agregue un nuevo módulo grande

---

# 5. Actualización del Índice

Cada vez que se agregue un nuevo documento:

1. agregarlo a SMARTWOD_PROJECT_INDEX.md
2. describir su propósito
3. indicar cuándo debe leerse

---

# 6. Buenas Prácticas

Mantener la documentación:

- clara
- concisa
- actualizada

Evitar duplicar información entre documentos.

Cada documento debe tener un propósito claro.

---

# 7. Beneficios

Mantener esta estructura de documentación permite:

- escalar el proyecto fácilmente
- facilitar mantenimiento
- mejorar la colaboración
- preservar conocimiento técnico
