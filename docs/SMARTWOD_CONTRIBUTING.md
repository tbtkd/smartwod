
# CONTRIBUTING.md
Guía de Contribución para SMARTWOD

Gracias por tu interés en contribuir al proyecto.

## Reglas generales

1. Mantener la arquitectura existente.
2. Evitar duplicación de lógica.
3. Documentar cualquier cambio importante.
4. Escribir código claro y legible.

## Estilo de código

- usar nombres descriptivos
- separar responsabilidades
- agregar comentarios en secciones críticas

## Pull Requests

Antes de enviar cambios:

- verificar que el proyecto compile
- probar manualmente el timer
- revisar que el historial funcione correctamente

## Reporte de errores

Al reportar un error incluir:

- tipo de entrenamiento
- pasos para reproducir el error
- comportamiento esperado
- comportamiento observado

Esto facilita la reproducción del problema.
